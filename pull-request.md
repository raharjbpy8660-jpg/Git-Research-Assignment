Pull request (PR)
• 	تعریف: در پلتفرم GitHub یک درخواست مشارکت برای ادغام تغییرات از یک شاخه (یا Fork) به شاخه پایه‌ی مخزن مقصد؛ همراه با بازبینی و بحث قبل از merge. هر کسی با دسترسی خواندن می‌تواند PR بسازد، ولی merge نیاز به دسترسی push دارد.
• 	کاربرد: بازبینی کد، تست‌های خودکار (CI)، مدیریت تعارض‌ها، حفظ کیفیت و پایداری شاخه اصلی.
