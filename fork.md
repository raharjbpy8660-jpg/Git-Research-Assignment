Fork
• 	تعریف: یک کپی کامل و مستقل از مخزن اصلی در حساب GitHub خودت می‌سازی؛ مناسب وقتی روی پروژه‌ای دسترسی نوشتن نداری و می‌خواهی پیشنهاد تغییر بدهی. Fork روی سرور انجام می‌شود (ویژگی GitHub)، نه با دستور خط فرمان.
• 	کاربرد: مشارکت در پروژه متن‌باز، آزمایش ایده‌ها بدون آسیب به مخزن اصلی، ارسال تغییرات از طریق Pull Request.
• 	زیرپوست (Under the hood): GitHub روی سرور یک مخزن جدید زیر حساب تو ایجاد می‌کند و رابطه با مخزن upstream را حفظ می‌کند تا بتوانی تغییرات upstream را همگام کنی.
