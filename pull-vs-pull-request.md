pull، pull request و push
• 	pull: دریافت و ادغام تغییرات از ریموت به محلی؛ فرمان Git در ترمینال.
• 	push: ارسال commitهای محلی به ریموت؛ نیاز به دسترسی نوشتن به مقصد.
• 	pull request: فرایند مشارکت روی GitHub برای پیشنهاد ادغام تغییرات از شاخه/فورک به شاخه مقصد؛ شامل بازبینی و سیاست‌های کیفیت.
• 	نکته جهت جریان: در PR عملاً از مالک مخزن مقصد می‌خواهی تغییرات تو را “pull” کند؛ به همین دلیل «pull request» نام گرفته، نه “push request”.
